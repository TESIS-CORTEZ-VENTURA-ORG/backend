## Modelo de datos (RLS FORCE en ambas tablas — riesgo R4)
- **`sales`** (`Sale`): `orderId` FK→`orders` **único** (un ticket por orden), `serie`, `number` (correlativo), `docType` (`boleta|factura`), `customer?`, `customerDoc?`, `subtotal`/`igv`/`total` `Decimal(12,2)`, `status` (`issued|void`, default `issued`), `voidReason?`, `issuedAt`. `@@unique([tenantId, serie, number])` (correlativo por tenant+serie) y `@@unique([orderId])`.
- **`payments`** (`Payment`): `saleId` FK→`sales` `ON DELETE CASCADE`, `method` (`cash|card|yape|plin`), `amount` `Decimal(12,2)`. **Varios pagos por venta** → pago mixto (HU-04-06).

Migración `sales_payments`; bloque RLS FORCE (`ENABLE`+`FORCE ROW LEVEL SECURITY` + policy `tenant_isolation`) **apendizado** para ambas tablas. `tenant_id` siempre desde el JWT; todo el acceso vía `runInTenant`.

## Cálculo de impuestos (precios INCLUYEN IGV)
Los precios de la carta ya incluyen IGV (mercado peruano). Para una orden:
- `total = Σ (unitPrice · qty)` de sus ítems vivos.
- `subtotal = total / (1 + igvRate)` (del `tenant.igvRate`, default `0.18`).
- `igv = total − subtotal`.

Toda la moneda se devuelve como **string** `.toFixed(2)` (PEN). Ej.: ítem 118.00 → subtotal 100.00, IGV 18.00, total 118.00.

## HU-04-01 · Generar pre-cuenta (preview, no persiste)
```gherkin
GIVEN orden con items
WHEN cajero solicita pre-cuenta
THEN se calcula subtotal, IGV (18%) y total
```
**`GET /api/orders/:id/pre-bill`** · `read Sale`. Calcula desde los ítems de la orden **sin persistir**. La orden no debe estar `paid` ni `void` (si no, 409). Devuelve:
```jsonc

```
> La propina sugerida (10%) del Gherkin se deja al frontend (config `payments.tipPct`); el backend no la persiste.

## HU-04-02 · Generar cuenta final (ticket) + HU-04-04/05/06 · Pagos
```gherkin
GIVEN pre-cuenta lista
WHEN cajero selecciona tipo (BOLETA/FACTURA) y datos del cliente
THEN se asigna serie y correlativo segun tipo AND el ticket queda listo
GIVEN ticket pendiente WHEN cajero ingresa pago(s) (efectivo / Yape / Plin / tarjeta)
THEN se valida que cubran el total AND el ticket pasa a PAID
```
**`POST /api/orders/:id/pay`** · `create Sale` · `@Audited('sale.pay')`. Body:
```jsonc
{ "payments": [{ "method": "cash|card|yape|plin", "amount": number>0 }],  // min 1
  "docType": "boleta|factura",  // default boleta
  "customer"?: string, "customerDoc"?: string }
```
Reglas (una sola transacción `runInTenant`):
1. La orden debe estar `open`/`served` y **no** pagada (si ya pagó/anuló → **409**).
2. `Σ amount ≥ total` (si no → **400**).
3. `serie = docType==='factura' ? 'F001' : 'B001'`; `number = (max number para ese tenant+serie) + 1` (correlativo, arranca en 1).
4. Crea `Sale` + N `Payment`; `order.status='paid'`; libera la mesa (`table.status='free'`).

Devuelve `{ order: OrderView, sale: SaleView }` (mismo contrato del mock BFF `orders/[id]/pay.post.ts` → adaptador trivial).
> Pago electrónico (HU-04-05): se **registra** el método; **no hay pasarela** (Yape/Plin/tarjeta sin gateway). La "referencia/nº de operación" del Gherkin no se modela en Inc 1.
> Efectivo (HU-04-04): el **vuelto** lo calcula el frontend (recibido − total); no se persiste.
> Mixto (HU-04-06): varios `Payment` cuya suma cubre el total.

## HU-04-07 · Anular ticket (con razón; manager/owner)
```gherkin
GIVEN ticket PAID WHEN cajero solicita anulación con razón (obligatoria)
THEN se requiere autorización del manager AND el ticket pasa a VOIDED
```
**`POST /api/sales/:id/void`** · `update Sale` (**manager/owner**; staff → **403**) · `@Audited('sale.void')`. Body `{ reason: string (min 1) }`. `issued → void`; razón obligatoria (si falta → **400**).
> **Fuera de alcance aquí:** reversar la orden/stock asociado (el Gherkin lo menciona; se documenta como follow-up — la orden queda `paid` y los movimientos de inventario no se revierten en Inc 1).

## Lecturas
- **`GET /api/sales`** · `read Sale` → `SaleView[]` (desc por `issuedAt`).
- **`GET /api/sales/:id`** · `read Sale` → `SaleView`.

### `SaleView` (espejo del `Sale` del frontend, moneda como string)
```jsonc
{ "id", "orderId", "serie", "number", "docType", "customer", "customerDoc",
  "date",          // = issuedAt ISO
  "tableLabel",    // `Mesa ${table.code}`
  "items": [{ "name", "qty", "unitPrice", "total" }],
  "subtotal", "igv", "total",
  "method",        // método del primer pago
  "payments": [{ "method", "amount" }],
  "status" }       // issued | void
```
